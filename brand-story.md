# Kenna Kai 品牌故事主线 — 军师型创新者

> 用品牌思维重构个人网站叙事

---

## 一、Kenna 的核心定位

**一句话定位**：
> "Kenna Kai — 创新业务的军师，用商业洞察连接资源与人性"

**身份标签栈**：
- **军师** — 不是冲锋的将军，是运筹帷幄的人
- **连接器** — 人、资源、机会之间的桥梁
- **洞察者** — 看见别人看不见的模式和人性
- **落地者** — 想法不只停在 PPT，能变成产品

---

## 二、品牌人格

### 人格画像：现代诸葛亮
- **不是最耀眼的人，但缺了不行**
- **话不多，但每句都在点上**
- **能看到三步之外**
- **手上有资源，脑子里有策略**
- **别人还在想，已经动手了**

### 品牌语气
- 沉稳但有锋芒
- 专业但不装腔
- 简洁但有分量
- 像一封精心写就的私信，不是公告

---

## 三、网站叙事结构（Story Arc）

### 第一章：「我是谁」— 定位锚点
**Hero 区**
- 大标题：不是名字，是身份宣言
- 副标题：价值主张，一句话说清楚为什么值得合作
- 视觉：有气场但不张扬，像军师出场

**文案方向**：
```
主标题：Strategic Mind, Creative Hand
副标题：I don't build products. I build the logic that makes products inevitable.

或中文：
主标题：谋定而后动
副标题：不是做产品的人，是让产品必然成功的人
```

---

### 第二章：「我的战场」— 能力版图
**About 区**
- 不是罗列技能，是展示"解决过什么复杂问题"
- 用场景化的方式呈现能力，不是标签堆砌

**能力维度**：

| 维度 | 核心能力 | 场景化表达 |
|------|---------|-----------|
| **商业洞察** | 看见别人看不见的模式 | "在噪音中听见信号" |
| **产品架构** | 从 0 到 1 的系统设计 | "把混沌变成路线图" |
| **人性理解** | 用户心理、团队动力 | "知道人在想什么，比知道产品怎么做更重要" |
| **资源整合** | 连接人、钱、技术 | "手上有资源，脑子里有策略" |
| **创新落地** | 从想法到 MVP | "想的和做的之间，没有墙" |

---

### 第三章：「我的战绩」— 项目叙事
**Projects 区**
- 每个项目讲一个"如何解决复杂问题"的故事
- 突出 Kenna 在其中的角色（不是执行者，是架构者）

**叙事模板**：
```
[项目名]
问题：当时面临什么复杂局面
洞察：看到了什么别人没看到的
策略：怎么拆解的
结果：最终效果
Kenna 的角色：军师 / 连接器 / 洞察者
```

---

### 第四章：「我的信条」— 价值观
**Creed / Philosophy 区**
- 不是空洞的 slogan，是真实的判断和选择
- 展示 Kenna 做决策的底层逻辑

**核心信条**：
1. **洞察先于行动** — "先看清棋盘，再落子"
2. **连接创造价值** — "一个人的能力有限，但连接起来的网络无限"
3. **人性是底层代码** — "所有商业问题，最终都是人的问题"
4. **极简是智慧** — "能一句话说清楚的，不用三句"
5. **落地检验想法** — "没有执行的想法只是幻想"

---

### 第五章：「和我一起」— 合作邀请
**Contact 区**
- 不是"hire me"，是"let's build something"的邀请
- 展示 Kenna 适合什么样的合作

**合作场景**：
- 创新业务从 0 到 1
- 复杂项目的策略拆解
- 资源整合与生态搭建
- 产品方向的迷雾中找路
